# PPT Design Reference Examples

## What Ahmed Considers "Good"
Based on 266+ Microsoft VBD training PPTs and Ahmed's feedback across sessions.

### Layout Rules
- **Breathing room**: minimum 0.5" padding around ALL elements
- **Visual density**: no more than 60% of slide area should contain elements
- **Section headers**: gradient or shape backgrounds, NEVER plain color + text
- **Icon usage**: every content slide should have at least one visual element (icon, diagram, image)
- **Text hierarchy**: clear distinction between titles (24-28pt), subtitles (18-20pt), body (12-14pt)

### Color Standards
- Primary: Microsoft blue (#0078D4), dark gray (#333333)
- Accent: teal (#008272), purple (#5C2D91), orange (#FF8C00)
- NEVER: bright red as primary, yellow text on white, low-contrast combinations
- Background: white or very light gray (#F5F5F5), NOT solid blue/dark backgrounds for content slides

### Anti-Patterns (Ahmed Has Rejected These)
1. ❌ Blue background with just white text = "lazy"
2. ❌ Text crammed edge-to-edge = "unreadable"
3. ❌ Slides with 5+ bullet points and nothing else = "a word document, not a slide"
4. ❌ Inconsistent header styling across sections
5. ❌ Generic stock photos that don't relate to the content
6. ❌ Overlapping elements of any kind

### What Gets Approved
- Clean layouts with 3-4 key points max per slide
- Icons from Fluent UI or similar Microsoft design language
- Data visualized (charts, diagrams) not just stated in text
- Consistent section transitions with visual identity
- "The first slide sets the mood — make it impressive"
